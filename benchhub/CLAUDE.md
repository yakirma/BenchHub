# benchhub/ — typed-contract package

Single source of truth for the typed contract. Imported by `app.py`, `metric_engine.py`, and the `benchhub-client` surface (`Client`, `SubmissionBuilder`, `BHDatasetCreator`). The root `CLAUDE.md` has the app-side summary; this file holds package internals (loads only when you edit `benchhub/`).

## Data types (`types.py`)
10 `DataType` subclasses: `Image`, `Mask`, `Depth`, `Audio`, `Text`, `BBoxes`, `Label`, `LabelList` (top-K, required `k`), `Scalar`, `Json`. Each has `encode()`/`decode()`/`validate()`/`visualize()` + the `DTYPES` registry. `/supported_types` is driven from `DTYPES` at request time so it can't drift from code.

**Metric authoring convention** (auto-derives from signatures): `def accuracy(gt: bh.Label, pred: bh.Label | bh.LabelList)` — annotations are parsed at save time into `GlobalMetric.input_kinds` (PEP 604 `|`-unions allowed); arg names `gt`/`pred`/`input` map to `input_roles` via heuristic. Runtime `isinstance` asserts live in the metric body; the engine injects `bh`/`benchhub` aliases into the exec scope.

## The typed pipeline (end-to-end)
1. **Admin uploads a typed dataset** via `POST /admin/import_typed_dataset` or `scripts/import_typed_dataset.py`. Format: a dir with `manifest.json` + `<field>/<sample>.<ext>`. Materialises Dataset + Sample + CustomField rows, copies file-backed kinds under `uploads/datasets/<id>/`; inline kinds (Scalar, Label) decode into `value_float`/`value_text`.
2. **LB declares its contract** via `Leaderboard.required_pred_fields_json` (`{name, kind, params, role}`; `role ∈ {input,gt,pred}`).
3. **Submitters use `benchhub-client`** — `bh.Client(token)` → `client.submission(lb_id).predict(sample, **kwargs).submit()`. Client validates each `DataType` at stage time, packs a ZIP matching the server's on-disk format, POSTs to `/api/submit/<lb_id>`.
4. **Server validates** the manifest against the contract (every pred name present, kinds match), writes Submission + CustomField pred rows, enqueues `tasks.process_submission`.
5. **Metric engine** builds per-sample context with both primitive (`gt_depth_pred`) AND typed (`__typed__gt_depth_pred`) entries. Metrics declaring `GlobalMetric.input_kinds` (non-empty JSON) get `bh.Depth` instances; legacy metrics keep the primitive. Five typed reference metrics seeded: `accuracy`, `rmse_depth`, `mae_depth`, `iou_mask`, `exact_match_text` (`scripts/seed_reference_metrics.py`, idempotent).

## Key files
- `types.py` — the `DataType` subclasses + `DTYPES`. Source of truth.
- `manifest.py` — manifest spec + `import_typed_dataset` + `import_typed_submission` + `check_submission_matches_contract`. `import_typed_dataset(..., preview_only=True)` routes vis modalities through the preview helpers, writes `.jpg`/`.png` instead of canonical extensions, sets `Dataset.preview_only=True`. **Sparse fields**: a field entry with `"optional": true` may miss its `<field>/<sample>.<ext>` file for some samples — the missing-file pre-flight skips it and the per-sample loop creates no CustomField for the gap (the sample is kept; *required* fields still hard-fail on any hole). `materialize_hf_to_typed_dir` auto-sets `optional` on any data-bearing field present for **some-but-not-all** rows (e.g. OpenFake's generation `prompt`, null for real images); a field null for **every** row stays required so a column/kind mis-map fails loudly instead of importing empty. Distinct from `tolerate_incomplete=True`, which *drops whole samples* missing any required field.
- `client.py` — `Client`, `SubmissionBuilder`, `FlaskTestClientTransport` (in-process transport tests use). `Client.iter_samples(lb_id, *, force_download=False)` pulls all file-backed inputs as ONE bulk ZIP from `/api/leaderboard/<id>/inputs.zip` (route `api_leaderboard_inputs_archive`, `ZIP_STORED`), extracts to `~/.cache/benchhub/<host>/lb_<id>/` (override via `$BENCHHUB_CACHE_DIR`), yields decoded `bh.<Kind>`. Cache keyed on `cache_token` from `/samples` (busts on re-materialise) + sorted `<field>/<sample>` list; `force_download=True` re-fetches. Masks pack raw `.classid.png` → decode to `bh.Mask`, not palette `bh.Image`. Falls back to per-sample `fetch_bytes` if the archive route 404s (older server). (Test cache isolation: see `tests/CLAUDE.md`.)
- `preview.py` — `image_preview`, `depth_preview` (turbo colormap), `mask_preview` (deterministic palette), `audio_preview` (waveform PNG); single dispatch via `render_preview(kind, payload)`.
- `hf_materialize.py` — `materialize_hf_to_typed_dir()` downloads an HF split (non-streaming `load_dataset`) into a typed-manifest staging dir. **Shard cap** (`shard_cap`): `-1` = whole split; `>0` = first N parquet shards; `0` = auto (download shards in order, reading each one's parquet-footer row count, until `sample_cap` rows are covered). `list_parquet_shards(repo, config_name=…)` resolves the `<config>/<split>-NNNNN-of-MMMMM[-NNNNN].parquet` layout (returns `{}` ⇒ caller falls back to a full load). Returns `shards_used`/`shards_total`; `tasks.run_hf_import` stashes them in `Dataset.source_metadata`. **Why it matters**: the non-streaming `load_dataset` pulls the *whole* split before `sample_cap` is applied, so a sample cap bounds *stored* rows, not the *download* — the shard cap is the only lever that bounds download bytes/time on huge repos (OpenFake `core/test` = 13 shards / 67.6 GB). `run_hf_import`/`run_file_tree_import` have **no Celery soft/hard time limit** (removed — a flat wall-clock cap was killing progressing imports; a wedged one holds a worker slot till restart, bounded by the non-admin one-import-at-a-time guard). Caveats: shards aren't guaranteed shuffled (capped slice can be class-skewed); the LB re-fetch's name↔row alignment only holds for head/full or content-named imports. **Classification coverage policy** (in `materialize_hf_to_typed_dir`, applies to BOTH the dataset preview tier AND per-LB materialisation since both call it): any *capped* import (`sample_cap>0`) with a `label`/`labellist` GT field is forced to `sampling='stratified'` and its cap raised to `max(2000, 10×n_classes)` (n_classes from the label field's `params['names']`), so every class is cached with ≥10 examples instead of head-sampling only the first few off a class-sorted split. `_pick_indices` stratified reads the label column in one `ds[col]` shot (no per-row image decode). `scripts/recache_classification.py` re-caches an existing dataset in place under this policy (swaps Sample/CustomField/DatasetField + previews; leaves LB-scoped GT rows untouched). A hidden-GT test split (all labels one value, e.g. `Bingsu/Human_Action_Recognition`) can't be fixed by re-sampling — re-point `source_metadata.split` to the labeled split (train) and re-cache.
- `lb_materialize.py` — `pick_samples()` (head/random/stratified) + `materialize_for_lb()` (re-runs `materialize_hf_to_typed_dir` at full res into a temp dir, status `ready`) + `materialized_or_preview_path()` (legacy file-path resolver: materialised wins, preview fallback). Wrapped by the Celery `tasks.materialize_leaderboard` so big imports don't block the request. **Decoupled LB eval set**: materialization now samples from the source split directly using LB-level `sample_cap`/`sampling`/`sampling_seed`/`shard_cap`/`split`/`config_name`, not from `dataset.samples` and not from the dataset import's `shards_used`. It writes LB-scoped GT `CustomField` rows (`leaderboard_id`, `sample_name`, `sample_id=NULL`, `submission_id=NULL`) plus full-res files under `uploads/lb_materializations/<lb_id>/`; `_iter_lb_eval_samples`, `/api/leaderboard/<id>/samples`, `/inputs.zip`, and comparison view synthesize virtual samples from those rows when present. This lets an LB over a preview-capped dataset evaluate samples that were never cached in the dataset preview tier.
- `file_tree_import.py` — the self-service file-tree importer engine (non-tabular HF repos, paired files, packed npz); routes `/import_from_files*` in `app.py`. **Source-agnostic**: `materialize_file_tree(spec, files, fetch, staging)` knows nothing about HF/Kaggle — `fetch(relpath)->localpath` is the only source hook. **Kaggle conversion loaders** added here (additive, see the `materialize_file_tree` elif chain + the `_parse_voc_xml`/`_coco_index`/`_rle_dims`/`_resolve_shared_index_samples` helpers): `rle` (RLE-in-CSV→Mask, column-major), `coco` (COCO JSON→coco_detections), `voc`/`yolo` (→bboxes), and a `csv`/`parquet`/`rle`/`coco` field marked `index:true` (rows/ids enumerate samples — pure-tabular has no file index). `_STAGE_EXT` gained `bboxes`/`coco_detections`→`.json`. Table-indexed samples carry a `_row` token so `sample_offset`/`cap` slicing doesn't desync row picking.
- **Kaggle importer** (the Kaggle analogue of the HF path — `docs/KAGGLE_IMPORT_PLAN.md` is the design, `KAGGLE_IMPORT_PROGRESS.md` the build log):
  - `kaggle_convert.py` — pure numpy/PIL converters (RLE↔labelmap, COCO RLE, bbox normalizers, palette/legend→labelmap, instance compositing, 16-bit depth→float). ⚠️ Kaggle RLE is **column-major (`order='F'`), 1-indexed** — C-order silently transposes every mask. Exhaustively tested (`tests/test_kaggle_convert.py`).
  - `kaggle_detect.py` — `detect_shape(files, peek=)` fingerprints an extracted tree → a `materialize_file_tree` spec + the **hidden-GT guard** (`partition_hidden_gt`: drops competition `test.csv`/`sample_submission`/`test/` from GT). Shapes A/A′/A″ tabular, C ImageFolder, D image+CSV, E COCO, F VOC, G YOLO, H paired mask, I RLE-CSV, N restoration pairs, O DICOM/NIfTI (flagged not-benchmarkable). `peek` can come from Kaggle's list-files `columns` metadata (no download needed for tabular detection).
  - `kaggle_client.py` — REST-over-`requests` (NO `kaggle` pip dep). **Two auth schemes, token-first**: a Bearer **access token** (`KGAT…`, from `$KAGGLE_API_TOKEN`/`$KAGGLE_ACCESS_TOKEN` or `~/.kaggle/access_token` — the kaggle CLI v1.8.0+ format, sent as `Authorization: Bearer`) via `resolve_access_token()`, else HTTP-Basic `(username,key)` from `$KAGGLE_USERNAME`/`$KAGGLE_KEY` or `~/.kaggle/kaggle.json` via `resolve_credentials()`. `from_env()`/`_ensure_resolved()` prefer the token; explicit constructor creds win over ambient. `view`/`list_files`(paginated)/`search`/`download`(whole-zip→cache dir keyed `owner__slug__vN`)/`fetch_factory`; 429+5xx backoff (injectable `sleeper`). **`classify_license(name)`** is the legal gate (pure, STRICT: NC/ND/GPL/Other/Unknown→restricted; fail-safe→False). **REST endpoints VERIFIED live (Bearer, 2026-06-08)**: files endpoint is `/datasets/list/{o}/{s}` (returns `datasetFiles`), NOT `/datasets/files/...` (404).   - `kaggle_search.py` — `search_datasets`/`card_summary`/`trending_by_domain` (1h TTL, mirrors `hf_search`), each threading a `KaggleClient` (Kaggle needs auth even to search); records carry the license verdict.
  - `tasks.py:run_kaggle_import` — mirrors `run_file_tree_import`: download cached zip → list as repo-relative → `materialize_file_tree` → quota → `import_typed_dataset(preview_only=True)` → stamp `source_kind='kaggle'`, `kaggle_slug`/`kaggle_version`/`license_name`/`license_redistributable`. ## User-registered data types (`DataTypeDef`) + the decode hook
- Register a new `kind` BenchHub doesn't ship (NIfTI, point clouds, EEG, …) via **`/datatypes/new`** (route `datatype_new`; `/supported_types` "Add a data type" links here) or `client.create_datatype(...)`. POST `/datatypes/create` (`create_datatype_web`) → back to `/datatypes/new` on error, `/supported_types` on success; `/datatypes` 302s to `/supported_types`.
- `DataTypeDef` columns: `name` (globally unique, shares the `DTYPES` namespace), `file_ext` (NULL ⇒ inline text), `viz_mime`, `visualize_code` (`def visualize(blob, params)->PIL.Image`), optional **`decode_code`** (`def decode(blob, params)->object`), `owner_user_id`, `visibility`. Storage is **bytes-verbatim** (encode=identity); `visualize`/`decode` run **only in the sandbox** (see `runner/CLAUDE.md`).
- **Decode hook = deserialize side of the contract.** With `decode_code` set, a consuming metric gets the decoded object (mirrors `bh.Depth.array`); absent ⇒ raw bytes. Wiring: `metric_engine.RegisteredBlob` (`kind,blob,params,decode_code`) is the carrier — `get_metric_context` emits it for any GT/input/pred CustomField whose `data_type` ∉ `DTYPES` (via `_registered_blob_for_cf`, lazy `from app import DataTypeDef`). Sandbox: `_jsonify_kwarg` → `{"__dtype__","decode","params","b64"}`, `runner/harness.py:_decode_arg` runs `decode` inside the metric's own container. In-process: `evaluate_dynamic_metric` resolves via `_resolve_registered_blob`.
- **Import admission**: `benchhub.manifest` can't see `DataTypeDef`, so `validate_manifest`/`load_manifest`/`expected_file_path`/`import_typed_dataset` take optional **`extra_kinds={name: file_ext}`** (accepted, stored verbatim, no preview). Server passes `app._registered_extra_kinds(owner_user_id)` (public + owner's own) at 4 sites: `_ingest_typed_dataset_zip`, `admin_import_typed_dataset` (request scope → `g`), `tasks.run_hf_import`/`tasks.run_file_tree_import` (Celery — **lazy-import** inside the task; do NOT add to top-level `from app import`, circular).
- **Registered-kind predictions (bytes-in).** Both GT/input AND pred support registered kinds. Submitter serializes their output; client packs **verbatim** — deliberately **no `encode` hook** (producer owns serialization; `decode` is the only hook since only the *server* deserializes to score). Client: `benchhub.RawPrediction(kind, data, *, file_ext=None, params=None)` (`.from_file(...)`); `SubmissionBuilder.predict(sample, field=RawPrediction(...))` packs bytes under the field's ext, which comes from the LB **contract** (`/contract` entries enriched with `file_ext` via `_kind_file_ext`) — so `set_contract()`/`fetch_contract()` (or explicit `file_ext=`) required or `build_zip` raises. Server: `validate_submission_manifest`/`import_typed_submission` take `extra_kinds`; submit route passes `_registered_extra_kinds(lb.owner_user_id)`. `check_submission_matches_contract` is kind-string only (no change); `_enforce_shape_constraint` skips registered kinds.
- A registered kind used by a public LB can't be deleted or made private (`_datatype_used_by_public_lb` guard).
