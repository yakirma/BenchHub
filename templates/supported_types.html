Supported types

How BenchHub recognizes, stores, and serves the fields in your datasets and submissions.

BenchHub auto-detects the type of each field from a combination of the file extension and (for .npz) the archive's key names. No reserved folder prefixes — name your folders whatever makes sense for your data, and the engine reads the type from what's inside. The same rules apply on the ground-truth side (a Dataset uploaded as a ZIP, or a HuggingFace dataset attachment) and on the prediction side (a Submission uploaded via the API). Every per-leaderboard Submission contract is just a filtered view of these rules — which folders that LB needs.

Reference table

Type File ext. Per-sample naming How it's stored / read
scalar .txt {{ '.txt' }} UTF-8 text containing a single number. Most often a float (0.873) or an integer class index. Parsed to float when handed to metric code.
text .txt {{ '.txt' }} UTF-8 string. Free-form — generated captions, sentiment labels (pos/neg), short classifier outputs. Passed to metric code as a Python str.
image .png / .jpg / .jpeg / .bmp / .tiff {{ '.' }} RGB or grayscale raster. Read as a PIL.Image when a metric or viz asks for it. The submit scaffold auto-converts numpy arrays / PIL Images to PNG HxWx3 uint8.
mask .png {{ '.png' }} Segmentation mask — either single-channel HxW (uint8/16 class IDs) or three-channel HxWx3 RGB-encoded classes. The engine inspects the shape and picks the right decode path.
depth .npz {{ '.npz' }} numpy .npz archive with one array under key depth (HxW float). The engine peeks at the keys to disambiguate from a histogram archive.
histogram .npz {{ '.npz' }} numpy .npz archive with two arrays: bins + counts. Detected by inspecting the archive's keys. Surfaces in the Comparison view as an inline chart and exposes a sub_entropy_<field> convenience value to metric code.
json .json {{ '.json' }} Arbitrary structured data, parsed to a Python dict. Useful for per-sample config or complex predicted objects (bboxes etc.).

Using data in metrics

Detected fields become arguments you map onto a metric function in the Leaderboard Settings. The dispatch table:

  • scalar / text → Python float / str passed by value.
  • image / mask → file path; load with PIL.Image.open(path).
  • depth → file path; load with np.load(path)['depth'].
  • histogram → pre-loaded dict with 'bins' + 'counts' arrays.
  • json → pre-loaded Python dict.

Every leaderboard page has a Submission contract widget that lists exactly which folders that LB requires — same rules as above, scoped to the metrics and visualizations attached to that LB.
