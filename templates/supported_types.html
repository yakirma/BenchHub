The data types BenchHub understands — inputs, ground-truth values, and predictions.

How a leaderboard declares its contract

Each leaderboard publishes a typed contract — the inputs the submitter receives, the predictions they must emit, and the ground-truth columns the server holds back for metric evaluation. The same JSON schema drives client-side construction, server-side validation, and the submission-contract widget on the leaderboard page. 

{
  "inputs": [
    {"name": "image", "kind": "image", "params": {}}
  ],
  "predictions": [
    {"name": "depth_pred", "kind": "depth", "params": {"unit": "meters"}}
  ],
  "ground_truth": [
    {"name": "depth_gt", "kind": "depth", "params": {"unit": "meters"}}
  ]
}

Submitting predictions

import benchhub as bh, numpy as np

client = bh.Client()                       # reads BENCHHUB_API_TOKEN
lb = client.leaderboard("nyu-depth-v2")

for sample in lb.samples():
    image = sample.inputs.image.array      # decoded uint8 (H,W,3)
    pred  = model.predict(image)           # your code
    sample.predict(depth=bh.Depth(pred, unit="meters"))

submission = lb.finish_submission(name="my-resnet50-v1")
print(submission.metrics)                  # {"rmse": 0.412, ...}
