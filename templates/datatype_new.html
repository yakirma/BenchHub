← back to Data types

Add a new kind BenchHub doesn't ship (NIfTI volumes, point clouds, EEG, …). The file you import is stored verbatim; you provide a visualize(blob, params) that renders one sample — it runs only in the hardened sandbox. Same thing from the client: client.create_datatype(...).

lowercase, 2–40 chars; joins the global kind namespace
file-backed bytes; blank = inline text
blob = the raw bytes of the stored file for one sample (exactly what was imported — e.g. a .nii.gz); params = that field's params dict. Parse blob with whatever library fits and return a PIL.Image. Runs in the sandbox (network-isolated, read-only), so heavy imports are fine.
Registers as private — flip it public on the Data types page to let others use it.
