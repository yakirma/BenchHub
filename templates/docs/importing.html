Importing datasets

Cache any HuggingFace {% if kaggle_import_enabled %}or Kaggle {% endif %}dataset into BenchHub — tabular, file-tree, or auto-detected — from one button.

One button, two paths

From Datasets → Add a dataset, paste an HF repo id and click Fetch & preview. BenchHub tries the tabular importer first (Croissant / parquet schema). If the repo isn't tabular — paired files, packed archives, video clips — it hands off automatically to the file-tree mapper. You can also switch to the mapper yourself from the preview (“file-tree importer” link) if the inferred mapping looks wrong.

Either way, the dataset is cached as the lightweight preview tier and defaults to private — publish it when you're ready.

Limits & safety. An import caches the full split (no row cap) but is bounded by your storage quota; non-admins run one import at a time; dataset loading scripts are never executed (parquet/files only); gated or private HF repos require a HuggingFace access token on your account — most benchmark datasets are public and import directly.

Tabular import

For datasets HuggingFace exposes as columns (Croissant or the datasets-server /info schema). The preview shows an editable table — one row per column:

  • Field name and kind (pre-inferred; override e.g. an image column that's really a mask);
  • params (e.g. a label vocabulary, depth unit);
  • source column (the HF column it reads from);
  • a split dropdown (with row counts), a max samples box (-1 = the whole split; defaults to 10,000 when the chosen split has more than 10,000 rows, otherwise the whole split), sampling (head / uniform / stratified) + seed, and an optional “use as sample name” column.
  • Max shards to download — shown only when the chosen split is stored as multiple parquet shards. A split downloads in full before max samples is applied, so a sample cap bounds what’s stored, not what’s downloaded; this caps the download itself. Pick All shards (whole split), Auto (download just enough shards, in order, to cover your max samples), or First N. It turns a giant split (e.g. 13 shards / 67 GB) into a few GB. Trade-off: shards aren’t guaranteed shuffled, so a capped slice can be skewed (e.g. one class dominating) — great for a quick test import, less so for a balanced benchmark. A leaderboard built from the dataset later re-fetches only those same shards, not the whole split.

Before committing, BenchHub screens the repo: gated/private without a token is blocked; non-parquet or JSON-fallback columns get a warning so you know they won't render or score as a typed kind. Your edits autosave as a draft — navigate away and a banner offers to restore them.

Sparse columns are fine. A column that's null for some rows (e.g. a generation prompt present only for synthetic images, or a caption that's missing on a few samples) imports cleanly — those samples simply carry no value for that field instead of failing the import. A column that's null for every sampled row is treated as a likely mapping mistake and fails loudly, so you can fix the kind or source column.

File-tree import

For repos that are a tree of files. Open it via /import_from_files (or the automatic hand-off). BenchHub lists the file tree and you describe how files become samples and fields.

Token patterns

You point each field at its files with a pattern containing {tokens}. A token matches exactly one path segment (it never spans /) and captures its value:

train/{seq}/normal/{id}.png
  matches  train/i_0/normal/1689562370.png   ->  seq=i_0, id=1689562370

The index modality — the first file field — enumerates your samples; every other field reuses the same token names to find its file for each sample. Special tokens:

  • {id} — the per-sample varying part (usually the filename). Also the default match key for csv/parquet and lets a zip/tar act as the index.
  • {ordinal} — a built-in: the sample's position within its group, in sorted order. Used by shared sources (json pointers, archive members).
  • {frame} — marks the per-frame token of a sequence field (samples are the groups; frames are gathered into a clip).

Describe the structure (the fast way)

The mapper analyzes the directory skeleton and shows one row per path level with example values and a role dropdown. Set what each level means and click Generate fields from structure to auto-build the field rows:

Level roleGenerates
idThe per-sample part → {id}.
modalityEach value is a separate field (image/, depth/, …).
propertyA class folder → an image field + a token label field.
splitA train/test partition → a {split} token wired to the subset filter.
groupA sequence/scene grouping → a {seq} token.
fixedA constant segment, kept literal.

Smart defaults pre-select the roles (the filename is id, modality-named folders are modality, arbitrary folders are property, …) — you just confirm or correct. You can always hand-edit the generated rows.

Loaders

Each field has a loader that says where its value lives:

LoaderUse it for
fileOne file per sample ({id}.png). The first file field is the index.
tokenThe value is a captured token — e.g. a class folder becomes a label (with a vocabulary built automatically).
npzA NumPy archive: a per-sample key, or one shared stacked array split along an axis.
jsonA JSON file + a dotted pointer (frames.{id}.pose, list.{ordinal}.q).
csv / parquetA table: pick a column, matched by an id column or by row order.
hdf5An .h5 dataset key; per-sample or a shared stacked array + axis.
zip / tarA container: point at a member inside (images/{id}.png). A zip/tar whose member uses {id} can be the index.
gzA single gzip-compressed file (e.g. {id}.json.gz), decompressed whole into the field value — no member pointer.
sequenceGroup frames into a clip (video): pattern uses {frame}; samples are the groups. See Data Types.
rleRun-length-encoded masks in a CSV (the dominant Kaggle segmentation format) → an integer-label-map mask. Column-major (Fortran) order, 1-indexed by default. Multiple rows per image composite into one map (per-class via class_column, else per-instance).
cocoA COCO detection JSON (images+annotations+categories) → coco_detections per image (bbox kept as COCO xywh).
vocPer-image Pascal VOC XML → bboxes (VOC's 1-indexed corners shifted to 0-indexed xyxy).
yoloPer-image YOLO TXT (class + normalized cxcywh) → bboxes (denormalized against the paired image's size, xyxy).

A csv/parquet or rle/coco field marked index enumerates the samples from the source itself — for pure-tabular datasets that have no per-file index modality (one row = one sample).

Examples

# Paired RGB + depth, depth packed as one stacked array per sequence
image : loader=file  pattern=train/{seq}/normal/{id}.png
depth : loader=npz   pattern=train/{seq}/normal/depth.npz  key=depth  shared=yes  axis=0

# Folder = class label (image classification)
image : loader=file   pattern={label}/{id}.png
label : loader=token  token=label                       (kind=label)

# A video clip per sequence
clip  : loader=sequence  pattern=clips/{id}/{frame}.png  (item_kind=image)

Very large repos — choose a split first

For a big repo (tens of GB or more), the mapper first shows a split / subfolder chooser instead of walking every file: pick a top-level folder (e.g. dataset/valid), Browse to drill in, or Preview & map to scope the preview to that subtree. Only the chosen subtree is listed — and only it gets imported. (A 21 TB, 250k-file medical repo that times out as a whole lists its valid split completely in a couple of seconds.) You can change the scope from the mapper at any time, or “Preview the whole repo as-is” to skip the chooser.

The mapper also lists files under a time budget so nothing can hang the page. If a scope is still enormous, the preview is built from a representative sample with a “very large” banner (your pattern still imports the full matching set at commit time); if even the sampler can't reach data files, the import is blocked with a pointer to narrow further, the tabular importer, or a data-type request. Repos whose files are an undecodable kind (e.g. .nii.gz NIfTI medical volumes) are flagged so you know they won't render or score.

Confirm before you commit

  • Decode preview — runs your mapping on the first sample and shows the decoded thumbnail/value per field, so you catch a wrong kind (the classic “that .npz is events, not depth” trap) before importing.
  • Field validation — missing required inputs are highlighted in red with a summary.
  • Draft — the whole mapping autosaves; leaving warns you, and returning offers to restore it.

Splitting & variants

  • Import only one split/subset — set “import only where token = value” (e.g. split = test).
  • Variant fan-out — set a token to split into separate datasets, one per value (e.g. a {quality} token with low/normal → <name>_low, <name>_normal).

Kaggle import

Open /import_from_kaggle (linked from Datasets → Add dataset). Search or paste a dataset ref (owner/slug), and BenchHub auto-detects the shape and pre-fills the field mapping — you review/correct it, optionally preview a decoded sample, then import. Browsing uses only cheap metadata; bytes download on preview/commit (whole archive, cached by slug+version).

Recognised shapes include single-table tabular classification/regression and text classification, ImageFolder (class dirs), image+CSV label maps, paired image/mask segmentation, restoration image pairs (dirty→clean), and the detection/segmentation annotation formats above (COCO, VOC, YOLO, RLE-in-CSV).

  • Hidden-GT guard — competition test.csv / sample_submission.csv / unlabeled test/ dirs have no labels, so they're excluded from ground truth; the eval split is built from labeled train data. A dataset with no usable GT is flagged not benchmarkable rather than importing an empty leaderboard.
  • License gate — only datasets under a redistributable license (CC0, CC-BY, CC-BY-SA, ODbL, PDDL/DbCL, CDLA-Permissive, U.S. Government Works, Apache/MIT/BSD) can be made public. Non-commercial (NC), no-derivatives (ND), GPL, “Other”, and “Unknown” are imported private to you and can't be published — caching and re-serving their bytes would be redistribution.
  • Access — mirrors the other importers: any signed-in user can import (lands private, charged to the private quota, one import at a time); admins' redistributable imports default to public. Needs a server-side Kaggle service account (KAGGLE_USERNAME/KAGGLE_KEY).
