1 · Tabular dataset → leaderboard → first submission

The fastest path: a HuggingFace dataset with columns (image + label), scored by accuracy.

  1. Import. Datasets → Add a dataset, paste e.g. uoft-cs/cifar10, click Fetch & preview. Confirm the inferred kinds (an image column and a label column), pick the test split, leave max samples at your taste, and commit. The dataset caches as a preview.
  2. Create a leaderboard. Open the dataset → Create leaderboard. Roles: image = input, label = gt. Prediction field: label_pred (kind label). Sample selection: 1,000 samples, stratified by label. Metric: accuracy, mapped gt→label, pred→label_pred. Create — materialization runs in the background.
  3. Submit. Grab a token from API tokens, then: 
    import benchhub as bh
client = bh.Client(token="bh_xxx")
LB = 7   # your leaderboard id (in the URL)

sub = client.submission(LB, name="my-classifier")
for name, inputs in client.iter_samples(LB):
    x = inputs["image"].array        # (32,32,3) uint8
    sub.predict(name, label_pred=bh.Label(int(my_model(x))))
sub.submit(description="baseline CNN")
  4. Compare. Your submission appears on the leaderboard with its accuracy + rank. Open the comparison view to see per-sample predictions vs GT.

2 · File-tree: paired RGB + depth

A repo laid out as train/<seq>/<frame>.png for RGB plus one packed depth.npz per sequence. Not tabular — use the file-tree mapper.

  1. Inspect. Fetch & preview hands off to the file-tree mapper automatically (no schema). It lists the tree and the path levels.
  2. Map fields. 
    image : loader=file  pattern=train/{seq}/normal/{id}.png   role=input  kind=image
depth : loader=npz   pattern=train/{seq}/normal/depth.npz  key=depth  shared=yes  axis=0
        role=gt  kind=depth
    The shared depth.npz holds all frames stacked along axis 0; BenchHub splits it and aligns frame N to the Nth image (sorted filename order).
  3. Decode preview. Click it — confirm the depth frame lines up with image 0 before committing.
  4. Leaderboard + metric. Create an LB with image=input, depth=gt, pred field depth_pred (kind depth), and bind rmse_depth (gt→depth, pred→depth_pred).
  5. Submit. 
    for name, inputs in client.iter_samples(LB):
    rgb = inputs["image"].array
    sub.predict(name, depth_pred=bh.Depth(my_depth_model(rgb), unit="meters"))
sub.submit()

3 · Folder-as-label image classification

A repo where the folder name is the class: Alex_Brush/000.png, Cookie/001.png, …

  1. Describe the structure. In the mapper, the “describe the structure” panel shows two levels: the class folder (default role property) and the filename (id). Click Generate fields from structure.
  2. It produces two rows automatically: 
    image : loader=file   pattern={label}/{id}.png   kind=image  role=input
label : loader=token  token=label                kind=label  role=gt
    The token loader turns each folder name into a label, building the class vocabulary across the dataset (so the UI shows Cookie, not an index).
  3. Leaderboard. pred field label_pred (kind label); bind accuracy. Submit as in tutorial 1.

4 · A video / clip (sequence) benchmark

Each sample is a clip of frames — e.g. clips/<id>/<frame>.png.

  1. Map a sequence field. 
    clip : loader=sequence  pattern=clips/{id}/{frame}.png   kind=sequence  role=input
    The {frame} token marks the per-frame part; each {id} becomes one sample whose frames are gathered into a clip. In the dataset and comparison views the clip renders as an mp4.
  2. Predicting a sequence. A sequence can also be a prediction kind. Read the clip and submit one back: 
    for name, clip in ((n, i["clip"]) for n, i in client.iter_samples(LB)):
    out = [bh.Image(my_frame_model(f.array)) for f in clip]   # clip is iterable
    sub.predict(name, clip_pred=bh.Sequence(out, item_kind="image", fps=10))
sub.submit()

5 · Write a metric + a confusion-matrix visualization

  1. Metric. On the Metrics page, add: 
    import benchhub as bh

def top1(gt: bh.Label, pred: bh.LabelList):
    return 1.0 if pred.values[0] == gt.value else 0.0
    Bind it to your leaderboard with gt→label, pred→label_pred, pooling = mean, higher-is-better.
  2. Visualization. A confusion_matrix aggregated viz ships ready to bind — map gt→label, pred→label_pred; its axes show class names automatically. Or write your own (see Visualizations).
  3. Re-score. Existing submissions don't need re-uploading — hit the recompute (↻) button on the leaderboard to apply the new metric.

6 · Submit from a script or Colab (reusable loop)

The submission loop is the same everywhere — iterate, run your model, predict, submit:

import os, benchhub as bh

client = bh.Client()                       # reads $BENCHHUB_API_TOKEN
LB = 42

# Inspect what the leaderboard expects:
print(client.leaderboard_contract(LB))     # [{name, kind, params, role}, ...]

sub = client.submission(LB, name="run-2026-06", link="https://github.com/me/model")
for name, inputs in client.iter_samples(LB):
    pred = my_model(inputs)                 # use inputs["<field>"]
    sub.predict(name, label_pred=bh.Label(int(pred)))
print(sub.submit())

Inputs are downloaded once and cached on disk (~/.cache/benchhub/, or $BENCHHUB_CACHE_DIR), so re-runs are offline. Pass force_download=True to iter_samples to re-fetch.

7 · Build a dataset in Python (no HuggingFace)

Have data already in memory or on disk? Skip the importer and push a typed dataset directly:

import benchhub as bh
client = bh.Client(token="bh_xxx")

ds = client.create_dataset("my-eval", visibility="private")
ds.add_field("image", kind="image", role="input")
ds.add_field("answer", kind="text",  role="gt")
for name, img, ans in load_my_data():
    ds.add_sample(name, image=bh.Image(img), answer=bh.Text(ans))
print(ds.create())

Then create a leaderboard on it exactly like an imported dataset.

8 · Variants & splits

When a file-tree repo encodes a quality or split in the path, e.g. train/<seq>/{quality}/<id>.png:

  • Import just one split/subset — set the mapper's “import only where token = value” to e.g. quality = normal. One dataset, only those samples.
  • Fan out into separate datasets — set the variant token to quality. BenchHub creates one dataset per value (<name>_low, <name>_normal), each importing only its slice.
Stuck on a specific repo's layout? File a feature request with the repo id and structure.
