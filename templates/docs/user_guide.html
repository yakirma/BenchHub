User Guide

A step-by-step guide to using Bench-Hub for your benchmarking needs.

1. Projects

A Project is the top-level container for your work. It groups together related leaderboards, datasets, and metrics.

  • Creating a Project: Click the icon on the Projects page. Give it a unique name (e.g., "DepthEstimation").
  • Cloning a Project: You can duplicate an existing project (including its leaderboards and metric settings) to quickly start a new iteration.
  • Switching Projects: Use the dropdown in the top navigation bar to switch between active projects.

2. Datasets (Ground Truth)

The Dataset serves as the "Ground Truth" for your benchmarks. Submissions are compared against this.

Note: Datasets are Global. They exist outside of projects and can be reused by any leaderboard in any project.
Structure Definition: Bench-Hub is largely agnostic to file formats, but creating a consistent directory structure is key.

Recommended Structure (ZIP file):

my_dataset.zip
├── sample_001.txt
├── sample_002.txt
└── ...

Or with folders:

my_dataset.zip
├── sample_001/
│   ├── depth.png
│   └── info.json
├── sample_002/
│   ├── depth.png
│   └── info.json
└── ...

When you upload this ZIP, Bench-Hub indexes the file paths. You will later use these paths ("fields") to map arguments to your metrics.

Data Types

Bench-Hub automatically detects file types (images, scalars, etc.). See the Supported Data Types page for details.

3. Leaderboards

A Leaderboard defines exactly what is being compared (which dataset) and how (which metrics).

  1. Navigate to the Leaderboards tab.
  2. Click Create Leaderboard.
  3. Select the Dataset you want to use as the Ground Truth.

Tip: You can Import Settings from another leaderboard during creation to copy all metric definitions and mappings.

Configuring Metrics

This is the most powerful part of Bench-Hub. Once your leaderboard is created, visit the Settings (Edit) page.

  • Add Metric: Choose a metric (e.g., L1_Error) from the library.
  • Argument Mapping: You must tell the system which files to feed into the metric function.
    • Arguments named gt or target usually map to a file in your Dataset.
    • Arguments named pred or submission map to a file in the Submission.

Example: If your metric is def calc_error(gt, pred):, you might map:

  • gt → Dataset File: sample_{id}/depth.png (selected as depth.png)
  • pred → Submission File: sample_{id}/result.png (selected as result.png)

4. Submissions

A Submission represents the output of an algorithm or system you are testing.

  • Upload: On the Leaderboard page, click Upload Submission.
  • Structure: The uploaded ZIP file must mirror the structure of the Dataset so that sample 001 in the submission can be matched to sample 001 in the dataset.

Once uploaded, the system automatically calculates all configured metrics for every sample.

5. Comparison & Analysis

Bench-Hub shines at deep analysis.

  • Leaderboard Table: Shows aggregated stats (Mean, Median, etc.). Click headers to sort.
  • Comparison View: Select multiple submissions and click Compare. This opens a side-by-side view where you can see:
    • Per-sample metric scores.
    • Custom Visualizations (e.g., Error Maps, Point Clouds).
    • Raw file contents (Images, Text, JSON).
