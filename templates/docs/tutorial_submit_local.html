Submit from a local script

Run the submission loop on your own machine or in CI — no notebook, no browser. Install the client, point it at a board by its handle, iterate the samples, run your model, and submit. The same shape works everywhere.

A submission is one model's predictions against a leaderboard's samples, packed and scored server-side. The benchhub-client package does the packing for you: you write a plain Python loop, it downloads the inputs, validates your predictions against the board's contract (the typed list of fields it expects), and POSTs the result. Because it's just a script, it drops straight into a Makefile, a cron job, or a CI step.

1 · Install

pip install benchhub-client

The package exposes everything under one import — import benchhub as bh gives you the client plus the typed bh.<Kind> constructors (bh.Image, bh.Label, bh.Depth, …).

2 · Get a token

Every request is authenticated with an API token. Generate one at Settings → account (the API tokens section, also reachable directly at /settings/api_tokens) and click Generate. Copy it once — it won't be shown again, and clicking Regenerate immediately invalidates the old token.

The client reads $BENCHHUB_API_TOKEN automatically, so the cleanest setup is to export it and never put the secret in source:

export BENCHHUB_API_TOKEN="<your-api-token>"

Then bh.Client() picks it up with no arguments. You can also pass it explicitly with bh.Client(token="<your-api-token>") — handy when juggling several accounts, but prefer the env var so the token never lands in a file or a shell history you might commit.

The base URL defaults to https://runbenchhub.com. Point the client at another instance by setting $BENCHHUB_BASE_URL.

3 · Two ways to get the code

You don't have to write the loop from scratch. Every board page has a Submit block with a tailored download:

  • Download script (.py) — a self-contained Python file, pre-filled for this board: it sets LEADERBOARD = "<owner>/<slug>" to the board's handle, wires up the right bh.<Kind>(…) prediction constructors for this board's pred fields, and leaves a my_model(inputs) stub for you to fill in. It reads BENCHHUB_API_TOKEN from the environment and raises if it's unset. Run it with python <board>_submit.py.
  • Write the loop yourself — the shape below is short enough to type by hand, and reusing it across boards is the point. Swap the handle and the bh.<Kind> constructors to match each board's contract.

If you'd rather run in a hosted GPU notebook, the same Submit block also offers a Colab path — see Submit from Colab.

4 · The reusable loop

This is the whole thing — reuse this shape for any board:

import os, benchhub as bh

client = bh.Client()                       # reads $BENCHHUB_API_TOKEN
lb = client.leaderboard("alice/cifar10-1k")  # the handle from the board page (an id works too)
print(lb.contract())                       # [{name, kind, params, role}, ...]

sub = lb.submission(name="run-2026-06", link="https://github.com/me/model")
for name, inputs in lb.iter_samples():
    img = inputs["image"].array             # decoded bh.Image -> NumPy array
    pred = my_model(img)
    sub.predict(name, label_pred=bh.Label(int(pred)))
print(sub.submit())

my_model is your code — replace it with whatever turns the decoded inputs into a prediction.

What each line is doing:

  • client.leaderboard("<owner>/<slug>") references the board by its handle — shown on the board page as “Client handle”. An integer id works anywhere a handle does.
  • lb.contract() returns the list of fields the board expects — each a {name, kind, params, role} dict. Read it once to confirm the role: "pred" field names and kinds you must produce.
  • lb.submission(name=…, link=…) stages a run; pass description= too if you want a blurb in the table. The link is the http(s) URL the submission name hyperlinks to.
  • lb.iter_samples() yields (name, inputs) for every sample in the board's materialized subset — the full-resolution samples the board actually scores on. inputs is a dict keyed by input-field name, and each value is an already-decoded typed instance — image/mask/depth arrive as bh.Image/bh.Mask/bh.Depth (use .array), audio as bh.Audio, sequence as an iterable bh.Sequence clip, and inline kinds (scalar/label/text/json/label_list) as the raw decoded value.
  • sub.predict(name, <field>_pred=bh.<Kind>(…)) stages one sample's typed prediction; the kind must match the contract or the client rejects it immediately.
  • sub.submit() packs everything into a ZIP, uploads it, and the server enqueues scoring.

5 · Caching and re-runs

File-backed inputs (images, masks, depth, audio, clips) download once as a single bulk archive and cache on disk under ~/.cache/benchhub/ (override with $BENCHHUB_CACHE_DIR). A second run of the same script reads straight from disk with no network — so iterating on my_model is offline and fast. When the board's samples change, or a download was interrupted, pass force_download=True to iter_samples to re-fetch:

for name, inputs in lb.iter_samples(force_download=True):
    ...

6 · Idempotency & CI

Because the token comes from an env var, the same script runs unattended in CI — set BENCHHUB_API_TOKEN as a secret and call python my_board_submit.py. To keep a re-run from creating a duplicate, guard on the name with lb.submission_exists(name):

import os

name = f"ci-{os.environ.get('GIT_SHA', 'local')}"
if lb.submission_exists(name):
    print(f"{name} already submitted; skipping")
else:
    sub = lb.submission(name=name, link="https://github.com/me/model")
    for sample_name, inputs in lb.iter_samples():
        sub.predict(sample_name, label_pred=bh.Label(int(my_model(inputs))))
    print(sub.submit())

7 · Run it

python my_board_submit.py

The submission appears on the board with its primary-metric score and rank as scoring runs (status Pending → Processing → Processed). Open the comparison view from the board to line up the ground truth against your predictions sample-by-sample — that's where the single number turns into insight about where your model fails. After a metric change you don't re-upload: hit the Recalculate (↻) control on the board to re-score the existing run.
