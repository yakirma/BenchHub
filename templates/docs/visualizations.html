Visualizations

Render an image to see a result — per sample, or summarized across a whole submission.

Like metrics, visualizations are Python functions stored in a reusable library and bound to a leaderboard with argument mappings. A visualization must return a PIL.Image (Pillow). To use matplotlib, render the figure to a PNG buffer and return Image.open(buf) (and plt.close(fig)). BenchHub caches the result and serves it into the dataset/comparison views.

Per-sample visualizations

Run on a single sample — e.g. overlay predicted boxes on the input image, or diff a predicted depth map against ground truth. The function receives the mapped fields for that sample and returns the rendered image.

def error_heatmap(gt, pred):
    import io, numpy as np, matplotlib.pyplot as plt
    from PIL import Image
    err = np.abs(np.asarray(gt) - np.asarray(pred))
    fig, ax = plt.subplots()
    ax.imshow(err, cmap="magma"); ax.axis("off")
    buf = io.BytesIO(); fig.savefig(buf, format="png", bbox_inches="tight")
    plt.close(fig); buf.seek(0)
    return Image.open(buf)          # a visualization returns a PIL Image

Aggregated visualizations

Run once per submission over all samples — e.g. a confusion matrix or a scatter of predicted vs true. The function receives the lists of values across samples. Aggregated visualizations get their own column and can be opened side-by-side across submissions in the comparison view.

def confusion_matrix(gt, pred, gt_names=None, pred_names=None):
    import io, json, numpy as np, matplotlib.pyplot as plt
    from PIL import Image
    # gt/pred are the per-sample label values across all samples; they may
    # arrive JSON-encoded ("3") or as top-K lists — normalize to an int.
    def idx(v):
        if isinstance(v, str): v = json.loads(v)
        return int(v[0] if isinstance(v, (list, tuple)) else v)
    g = [idx(x) for x in gt]; p = [idx(x) for x in pred]
    n = max(g + p) + 1
    m = np.zeros((n, n), dtype=int)
    for a, b in zip(g, p):
        m[a, b] += 1
    fig, ax = plt.subplots()
    ax.imshow(m)
    labels = gt_names or [str(i) for i in range(n)]
    ax.set_xticks(range(n)); ax.set_yticks(range(n))
    ax.set_xticklabels([f"{nm} ({i})" for i, nm in enumerate(labels)], rotation=90)
    ax.set_yticklabels([f"{nm} ({i})" for i, nm in enumerate(labels)])
    ax.set_xlabel("predicted"); ax.set_ylabel("ground truth")
    buf = io.BytesIO(); fig.savefig(buf, format="png", bbox_inches="tight")
    plt.close(fig); buf.seek(0)
    return Image.open(buf)
Class names for free. When a mapped argument is a label field, BenchHub also injects a matching <arg>_names argument holding that field's class-name vocabulary — so a confusion matrix can label its axes with real class names (cat (3)) instead of bare indices. A ready-made confusion_matrix visualization ships in the library.

Depth colormaps

Depth fields are stored once and recolored at view time. The comparison and dataset views expose a colormap selector on each depth column — turbo, viridis, magma, inferno, plasma, jet, gray, and a Sobel-based surface-normal view — served from /api/gt_viz/<lb>/<col>/<sample>?cmap=. You don't author this; it's built in for the depth kind.

Authoring & binding

  • Write and edit on the Visualizations page (Python source, same DLP-safe upload + public/private visibility + per-owner name rules as metrics).
  • Bind to a leaderboard on its Edit page, mapping each argument to a field. The rendered image is cached; aggregated ones precompute when a submission is processed.
