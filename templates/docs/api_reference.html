API & client reference

The benchhub-client Python surface and the underlying HTTP endpoints.

Authentication

Programmatic calls authenticate with a Bearer token from /settings/api_tokens (regenerate or revoke there). Send it as Authorization: Bearer <token>, or let the client read $BENCHHUB_API_TOKEN. Verify a token with GET /api/whoami.

Python client

import benchhub as bh
client = bh.Client(token="<your-api-token>", base_url="https://runbenchhub.com")
#                  ^ or $BENCHHUB_API_TOKEN          ^ or $BENCHHUB_BASE_URL (default)
lb = client.leaderboard("alice/cifar10-1k")   # reference a board by handle (an id works too)

Client

A ref is a board's <owner>/<slug> handle (shown on the board page as “Client handle”), a bare unambiguous slug, or an integer id — all interchangeable.

leaderboard(ref)Recommended entry point. Resolve a board and return a LeaderboardHandle whose methods carry the id for you.
iter_samples(ref, *, force_download=False)Yield (sample_name, inputs); inputs are decoded bh.<Kind> instances. File-backed inputs are bulk-downloaded once and cached.
leaderboard_contract(ref)The prediction contract: a list of {name, kind, params, role}.
submission(ref, name=None, *, description=None, link=None)Open a SubmissionBuilder.
submission_exists(ref, name)Whether a submission of that name already exists.
create_dataset(name, *, visibility="public")Open a BHDatasetCreator.

LeaderboardHandle (from client.leaderboard(ref))

.id / .ref / .name / .slug / .ownerThe resolved board's id, <owner>/<slug> handle, and metadata.
.iter_samples(*, force_download=False)Same as Client.iter_samples, no id needed.
.submission(name=None, *, description=None, link=None)Open a SubmissionBuilder for this board.
.contract() / .submissions() / .submission_exists(name)The board's contract, its submissions, and a name-exists check.

SubmissionBuilder

predict(sample_name, **fields)Stage typed predictions for a sample; each value is a bh.<Kind> (validated immediately).
submit(name=None, *, description=None, link=None)Pack + upload; returns the server response.
build_zip()The submission ZIP bytes (for inspection / manual upload).
fetch_contract() / set_contract(c)Load the LB contract for local shape checks.
set_input_shape(sample, **fld=(H,W))Feed input shapes so shape_match preds are checked before upload.

BHDatasetCreator

Build a dataset programmatically and upload it in one call:

ds = client.create_dataset("my-bench", visibility="private")
ds.add_field("image", kind="image", role="input")
ds.add_field("label", kind="label", role="gt",
             params={"names": ["cat", "dog"]})
for name, img, y in my_samples:
    ds.add_sample(name, image=bh.Image(img), label=bh.Label(y))
ds.create()

add_field(name, kind, role, params=...) declares the schema (optional — it can be inferred from the first add_sample); add_sample(name, **typed_values) stages one sample; create() uploads the package.

HTTP endpoints

The client wraps these; use them directly if you're not on Python.

Leaderboard data
GET /api/leaderboard/resolve?ref=<owner/slug>Resolve a handle (or id) to {id, name, slug, owner, ref}. What client.leaderboard() calls.
GET /api/leaderboard/<id>/infoMetadata. Also /api/leaderboard/by_name/<name>/info.
GET /api/leaderboard/<id>/contractThe prediction contract.
GET /api/leaderboard/<id>/samplesPer-sample inputs (inline values + URLs for file-backed fields) + a cache token.
GET /api/leaderboard/<id>/inputs.zipOne ZIP of all file-backed inputs, <field>/<sample>.<ext>.
GET /api/leaderboard/<id>/submissionsThe scored submissions.
Submitting & datasets
POST /api/submit/<lb_id>Multipart submission ZIP (manifest.json + <field>/<sample>.<ext>). Validated against the contract, then scored async.
POST /api/datasetsMultipart typed-dataset ZIP (the BHDatasetCreator target).
POST /api/leaderboard/<id>/recalculate_asyncRe-score submissions (e.g. after a metric change).
Rendering
GET /api/viz/<cf_id>Render any typed field's value (image PNG, depth colormap, mask palette, an mp4 for a sequence, …).
GET /api/gt_viz/<lb>/<col>/<sample>?cmap=Recolor a depth field with a chosen colormap.

For the end-to-end submission flow see Submit Predictions; for typed constructors see Data Types.
