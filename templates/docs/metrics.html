Writing Custom Metrics

Define how your submissions are evaluated using Python code.

Overview

Bench-Hub allows you to define metrics using Python. These snippets are executed safely to compute scores for your data.

Available Libraries

The environment comes pre-loaded with standard scientific libraries:

  • numpy (as np)
  • math
  • json

Metric Types

1. Per-Sample Metrics

These metrics calculate a score for each individual sample (e.g., L1 error for a single image).

Signature
def metric_name(arg1, arg2, ...):
    # Process inputs
    return float_value
Example: L1 Error
def L1(pred, gt):
    import numpy as np
    # Bench-Hub passes inputs as parsed values if possible, 
    # but robust code handles lists/arrays just in case.
    if isinstance(pred, (float, int)):
        pred = np.array([pred])
    if isinstance(gt, (float, int)):
        gt = np.array([gt])
        
    return np.linalg.norm(pred - gt, ord=1)
2. Aggregated Metrics

These metrics calculate a single score for the entire dataset (e.g., F1 Score, Total Accuracy). They receive lists of values.

Signature
def metric_name(arg1_list, arg2_list, ...):
    # Inputs are lists of all samples
    return float_value
Example: Precision
def precision(tp_list, fp_list):
    # Filter out None values (failed samples)
    tps = [x for x in tp_list if x is not None]
    fps = [x for x in fp_list if x is not None]
    
    total_tp = sum(tps)
    total_fp = sum(fps)
    
    if total_tp + total_fp == 0:
        return 0.0
    return total_tp / (total_tp + total_fp)

Best Practices

  • Handle Nones: Inputs might be None if a sample failed to load or a dependency is missing. Always check for validity.
  • Return simple types: Return python float or int. Numpy scalars should be cast (float(val)).
  • Dependencies: You can input the result of other metrics! Just map the argument to "Per-Sample Metric" in the leaderboard settings.
