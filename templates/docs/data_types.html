Supported Data Types

Understanding how Bench-Hub recognizes and parses your data.

Bench-Hub is designed to be flexible. It automatically detects the type of data in your Datasets and Submissions based on file extensions and folder naming conventions.

Note: Files are indexed when you upload a ZIP. You can view recognized fields in the "Dataset View" or "Submission View".

Recognition Rules

Type Recognition Rule Usage & details
Image Extensions: .png, .jpg, .jpeg, .bmp, .tiff

Detected as an image path.

Used in Visualizations (converted to PIL Image).
Scalar Extension: .txt
(content must be a single number)

Parsed as a python float.

Used in Metrics or as simple values in the Comparison table.
JSON Extension: .json

Parsed as a python dict.

Useful for complex data structures or configurations.
Histogram .npz file inside a folder starting with hist_ (or named raw_histogram)

Parsed as numpy arrays (bins, counts).

Used for Entropy calculations and Histogram plots.
Depth Map .npz file inside `raw_` folder, named sample_WxH.npz

Specific to Depth Benchmarking.

Contains raw depth arrays.

Using Data in Metrics

Once detected, these files become "Arguments" you can map in the Leaderboard Settings.

  • Scalars are passed directly as numbers.
  • Images are passed as file paths (use PIL.Image.open()).
  • JSON is passed as a dictionary.
  • Arrays (Histogram/Depth) need custom loading usually, but Histograms are pre-processed into dictionaries for some legacy metrics.
