Submitting predictions

Pull a leaderboard's inputs, run your model, and submit typed predictions with the benchhub-client Python package.

1. Install & authenticate

pip install benchhub-client

Grab an API token from Settings → API tokens. Pass it to the client, or set BENCHHUB_API_TOKEN in your environment (the base URL defaults to https://runbenchhub.com and can be overridden with BENCHHUB_BASE_URL).

import benchhub as bh

client = bh.Client(token="<your-api-token>")        # or rely on $BENCHHUB_API_TOKEN

2. The submission loop

The whole flow is: iterate the leaderboard's samples, run your model on each one's inputs, stage a typed prediction, then submit.

import benchhub as bh
import numpy as np

LB_ID = 42
client = bh.Client(token="<your-api-token>")
sub = client.submission(LB_ID, name="my-model-v1")

for sample_name, inputs in client.iter_samples(LB_ID):
    image = inputs["image"].array          # a decoded bh.Image -> (H,W,3) uint8
    pred = my_model(image)                 # your code; returns a class index, say
    sub.predict(sample_name, label_pred=bh.Label(int(pred)))

result = sub.submit(description="ResNet-50, 50 epochs",
                    link="https://github.com/me/my-model")
print(result)                             # -> {'submission_id': ..., ...}

3. Reading inputs — iter_samples

client.iter_samples(leaderboard_id) yields (sample_name, inputs) for every sample in the leaderboard's materialized subset. inputs is a dict keyed by input-field name; each value is an already-decoded typed instance:

  • image/mask/depth → bh.Image/bh.Mask/bh.Depth (use .array);
  • audio → bh.Audio;
  • sequence → bh.Sequence, an iterable clip — for frame in inputs["clip"]: frame.array;
  • inline kinds (scalar/label/text/ json/label_list) → the raw decoded value.

File-backed inputs are downloaded once as a single bulk archive and cached on disk (under ~/.cache/benchhub/, override with $BENCHHUB_CACHE_DIR), so a re-run reads from disk with no network. Pass force_download=True to re-fetch.

4. Staging predictions — predict()

sub.predict(sample_name, field=value, ...) stages one or more typed predictions for a sample. Each value must be a bh.<Kind> instance matching the leaderboard's contract; the client validates it immediately.

# classification
sub.predict(name, label_pred=bh.Label(3))

# dense prediction (depth / segmentation)
sub.predict(name, depth_pred=bh.Depth(pred_depth))
sub.predict(name, mask_pred=bh.Mask(pred_seg, num_classes=21))

# a predicted video clip (sequence)
frames = [bh.Image(f) for f in rendered_frames]
sub.predict(name, clip_pred=bh.Sequence(frames, item_kind="image", fps=10))

5. Matching the contract

To see exactly what a leaderboard expects, fetch its contract:

contract = client.leaderboard_contract(LB_ID)
# -> [{"name": "label_pred", "kind": "label", "params": {...}, "role": "pred"}, ...]

Every pred field in the contract must be produced for every sample. For spatial predictions some leaderboards require the prediction's shape to match an input field's shape (params.shape_match) or a fixed shape (params.shape); the client can enforce this locally before upload if you call sub.set_contract(contract) and feed input shapes via sub.set_input_shape(...) — otherwise the server checks at eval time.

6. Submitting

sub.submit(name=None, description=None, link=None) packs the staged predictions into a ZIP matching the on-disk layout (<field>/<sample>.<ext> + a manifest.json), POSTs it to /api/submit/<lb_id>, and the server validates it against the contract and enqueues scoring. The leaderboard updates as metrics finish (status Pending → Processing → Processed).

  • description — a blurb shown in the submission table.
  • link — an http(s) URL the submission name hyperlinks to (repo, model card).

7. After submitting

  • Your submission appears on the leaderboard table with its primary-metric score and rank.
  • Open the comparison view to inspect predictions vs ground truth sample-by-sample (images, depth colormaps, masks, video clips, per-sample metric values).
  • The submission's owner (or an admin) can re-score it with the recompute (↻) button — e.g. after a metric is added or changed — without re-uploading.
Prefer raw HTTP? POST a multipart ZIP (the same manifest.json + <field>/<sample>.<ext> layout) to /api/submit/<lb_id> with an Authorization: Bearer <token> header. See the API reference.
